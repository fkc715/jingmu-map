# Product Notes

## Working Name

Jingmu Map, Chinese name: 京幕地图.

The name combines Beijing, theatre curtains, and map exploration. It keeps the project close to the original idea: a small, personal map for discovering and recording theatre visits in Beijing.

## Core Idea

Jingmu Map is a personal Beijing theatre map and theatre-going journal. It helps the user:

- Browse theatres by Beijing district
- Open a theatre card from the map
- Record personal theatre visits
- Upload personal ticket cards, posters, or related images
- Count theatre visits over time
- Generate future monthly or yearly theatre-going summaries

## Version 1.0 Scope

Version 1.0 should stay small and focused:

- Draw a minimal Beijing administrative district map
- Mark selected theatre locations
- Show a theatre card after clicking a theatre marker
- Add simple click, highlight, or card transition animations
- Store theatre seed data in a structured format
- Prepare the project for both web and WeChat mini program development

Version 1.0 should avoid user uploads, public comments, social features, ticketing, and third-party event data aggregation.

## Version 1.1 Scope

Version 1.1 can add private theatre-going records:

- User login
- Personal theatre visit records
- Private calendar view
- Upload ticket cards, theatre posters, or related images
- Link each record to a theatre
- Count how many times a user has visited each theatre

Uploaded content should be treated as private by default. If the mini program version supports uploads, the backend should run image safety checks before displaying uploaded images.

## Version 1.2 Scope

Version 1.2 can add summary and report features:

- Yearly theatre-going report
- Most visited theatres
- Most visited districts
- Monthly visit timeline
- Poster or ticket collage
- Shareable report image generated by the user

## Product Boundaries

To keep the project easier to launch and review, the product should not initially provide:

- Ticket sales
- Ticket exchange or resale
- Public theatre reviews
- Public image feeds
- Public social posting
- Crawled event schedules from third-party platforms
- Unverified venue or event claims

The first public version should feel like a personal utility, not a public media or ticketing platform.

## WeChat Mini Program Considerations

Before publishing the mini program, the project will likely need:

- A registered WeChat mini program account
- Mini program filing and review preparation
- A privacy policy
- A user agreement
- HTTPS backend domain configuration
- Configured legal request and upload domains
- Content safety checks for uploaded images
- A way for users to delete their own data
- Basic admin ability to review or remove problematic content if public features are ever added

The safest early direction is to keep all uploaded images and records private to the user.
