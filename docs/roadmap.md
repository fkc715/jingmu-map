# Roadmap

## 0.1 Planning and Prototype

Goal: create the project foundation and validate the map-first experience.

- Create initial project documents
- Decide the first frontend architecture
- Prepare theatre seed data format
- Build a static Beijing district map prototype
- Add sample theatre markers
- Add a clickable theatre card
- Test the prototype on desktop and mobile screen sizes

## 1.0 Minimal Theatre Map

Goal: build a usable first version of the Beijing theatre map.

- Implement web frontend
- Implement mini program frontend or Taro cross-platform version
- Implement Go API for theatre data
- Add structured theatre data
- Add district filtering
- Add theatre marker interactions
- Add theatre detail cards
- Add basic responsive layout
- Deploy a web preview

Success condition: a user can open the app, browse Beijing theatres on a minimal map, click a theatre, and understand where it is and what it is.

## 1.1 Personal Theatre Calendar

Goal: let users record private theatre-going memories.

- Add user identity
- Add personal visit records
- Add calendar view
- Add image upload for tickets, posters, or related images
- Add image safety check workflow
- Link each record to a theatre
- Show visit counts on theatre cards
- Add data deletion flow for user records

Success condition: a user can record a theatre visit, attach an image, and see the theatre visit count update.

## 1.2 Theatre Report

Goal: turn private records into useful summaries.

- Add yearly statistics
- Add most visited theatre ranking
- Add district distribution
- Add monthly timeline
- Add generated share image
- Add simple report export or sharing flow

Success condition: a user can generate a personal yearly theatre-going report based on their own records.

## Later Ideas

These ideas should wait until the core private product is stable:

- Theatre tag system
- Seat view notes
- Transportation notes
- Manual event records
- Friend-only sharing
- Public recommendations
- Admin content review tools
